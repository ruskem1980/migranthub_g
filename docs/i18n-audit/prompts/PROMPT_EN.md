# Задача: Аудит локализации — Английский язык (en)

## Контекст

Проект MigrantHub — приложение для мигрантов в России.
- Файл локализации: `apps/frontend/src/locales/en.json` (~2405 строк)
- Английский — эталонный файл структуры для всех языков
- Используется как fallback и для англоговорящих пользователей

## Цель

1. Проверить профессиональность и корректность английских текстов
2. Проверить применение всех ключей в компонентах
3. Найти неиспользуемые ключи
4. Исправить найденные проблемы
5. Создать отчёт об изменениях

## Критерии качества перевода

### Профессиональность
- Корректная юридическая терминология на английском
- Нет калькированных переводов с русского
- Естественные формулировки для носителя английского
- Консистентность терминологии во всём файле

### Стилистика
- Американский английский (US English)
- Понятный язык без жаргона
- Краткость (UI-тексты должны быть лаконичными)
- Без опечаток и грамматических ошибок
- Title Case для заголовков, Sentence case для описаний

### Технические требования
- Все плейсхолдеры {{variable}} корректны
- Нет дублирующихся ключей
- JSON-синтаксис валиден

## Шаги

### 1. Валидация структуры
- Проверить JSON на синтаксические ошибки
- Найти дублирующиеся ключи
- Проверить вложенность

### 2. Проверка качества текстов
Проверить по секциям:
- `common` — общие элементы UI
- `auth`, `login`, `recovery` — авторизация
- `documents` — документы
- `services` — сервисы
- `exam` — экзамен
- `sos` — экстренная помощь
- `wizard` — визард легализации
- И все остальные секции

### 3. Проверка использования в компонентах
Найти все вызовы `t('ключ')` в tsx-файлах и проверить:
- Ключ существует в en.json
- Определить неиспользуемые ключи

### 4. Исправления
Внести корректировки в файл `en.json`

### 5. Отчёт
Создать `docs/i18n-audit/reports/REPORT_EN.md`

## Файлы для работы

- `apps/frontend/src/locales/en.json` — проверить и исправить
- `apps/frontend/src/**/*.tsx` — проверить использование
