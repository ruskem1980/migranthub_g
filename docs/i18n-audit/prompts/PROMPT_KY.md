# Задача: Аудит локализации — Киргизский язык (ky)

## Контекст

Проект MigrantHub — приложение для мигрантов в России.
- Целевая аудитория: мигранты из Киргизии
- Файл локализации: `apps/frontend/src/locales/ky.json` (~2398 строк)
- Эталон структуры: `apps/frontend/src/locales/en.json`
- Киргизский — важный язык для ЦА из Киргизии

## Цель

1. Проверить профессиональность и корректность переводов на киргизский
2. Проверить применение всех ключей в компонентах
3. Исправить найденные проблемы
4. Создать отчёт об изменениях

## Критерии качества перевода

### Профессиональность
- Юридическая терминология должна быть точной
- Нет машинного/калькированного перевода
- Естественные формулировки для носителя киргизского языка
- Консистентность терминологии во всём файле
- Использование кириллицы (официальный алфавит Киргизии)

### Стилистика
- Уважительное обращение к пользователю
- Понятный язык
- Краткость (UI-тексты должны быть лаконичными)
- Без опечаток и грамматических ошибок

### Специфика киргизского
- Правильное использование аффиксов
- Корректный порядок слов
- Уместное использование заимствований из русского (для миграционных терминов)
- Учёт диалектных особенностей (северный/южный)

## Шаги

### 1. Анализ структуры
- Сравнить ключи ky.json и en.json
- Найти отсутствующие ключи
- Найти лишние ключи

### 2. Проверка качества перевода
Проверить по секциям:
- `common` — общие элементы UI
- `auth`, `login`, `recovery` — авторизация
- `documents` — документы
- `services` — сервисы
- `exam` — экзамен
- `sos` — экстренная помощь
- `wizard` — визард легализации
- И все остальные секции

### 3. Проверка использования в компонентах
Найти все вызовы `t('ключ')` и проверить:
- Ключ существует в ky.json
- Перевод уместен в контексте компонента

### 4. Исправления
Внести корректировки прямо в файл `ky.json`

### 5. Отчёт
Создать `docs/i18n-audit/reports/REPORT_KY.md`

## Файлы для работы

- `apps/frontend/src/locales/ky.json` — проверить и исправить
- `apps/frontend/src/locales/en.json` — эталон структуры
- `apps/frontend/src/**/*.tsx` — проверить использование
